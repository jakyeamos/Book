<task type="auto">
  <name>Task 3: Write validate-chapters.cjs validation script</name>
  <files>tools/validate-chapters.cjs</files>
  <action>
    Create `tools/validate-chapters.cjs` as a CommonJS Node.js script. This is the test harness for all subsequent plans.

    The script accepts CLI flags and exits 0 on pass, non-zero on failure.

    Implement these modes:

    **`--check chNN [chMM ...]`** (one or more chapter slugs):
    For each slug:
    - Confirm `chapters/{slug}.html` exists.
    - Read its contents and confirm it contains `<div class="chapter" id="chapter`.
    - Confirm it contains at least one element with `class="chapter-title"`.
    - Confirm all `id=` attributes on `<section>` elements use a per-chapter prefix format (e.g., `ch1-s`, `ch2-s`, `ch15-s`) — NOT bare `section1`, `section2`, etc. Use a regex: `/id="section\d+"/` — if found, fail with "bare section ID detected".
    - Print `[PASS] {slug}` or `[FAIL] {slug}: {reason}`.

    **`--manifest`**:
    - Read `chapters/index.json`. If absent or not valid JSON, fail.
    - Confirm it is an array.
    - For each entry: confirm `id`, `file`, `title`, `number` fields exist.
    - Confirm the file path (`entry.file`) exists on disk (use `fs.existsSync`).
    - Print pass/fail per entry; exit non-zero if any fail.

    **`--lint-formatting`**:
    - For each `.html` file in `chapters/` (not in `chapters/raw/`):
    - Scan for `style="` attribute on any tag inside a `<div class="chapter">` block.
    - If found, print `[FAIL] {filename}: inline style found on line N`.
    - Exit non-zero if any violations found.

    **`--full`**:
    - Run `--manifest` first.
    - Then run `--check` for all chapter IDs found in `chapters/index.json`.
    - Then run `--lint-formatting`.
    - Print a summary: `N passed, M failed`.
    - Exit non-zero if any step failed.

    Implementation note: Use only Node.js built-ins (`fs`, `path`). No external dependencies beyond what is already in package.json.

    Edge case: If `chapters/index.json` does not yet exist, `--full` should print `[SKIP] manifest not yet created` and proceed without failing (chapters are still being authored). `--manifest` alone should fail if the file is missing.
  </action>
